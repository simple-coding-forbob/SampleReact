# 리액트 기본
## SSR vs CSR
## SSR: (Server Side Rendering), 서버에서 html 을 만들어서 웹브라우저로 전송하는 기술
   예) jsp 등
## CSR: (Client Side Rendering), 서버에서는 데이터만 보내고 웹브라우저에서 js 로 페이지를 만드는 기술
   예) react, vue, angular 등
## 리액트 내부 기능
 - 페이지의 특정 부분 갱신 속도가 빠릅니다.(가상 DOM과 바인딩 기술)
 예) JSP  : 태그를 찾아 값을 수정해야합니다.(태그 찾는 시간 걸림)
     react: 처음 1번은 전체 페이지를 읽습니다.(이전 페이지로 저장해둠(가상DOM))
            이후에 페이지 값이 변경되면 이전 페이지와 비교해서 다른부분만 빠르게 바꿉니다.
            (찾지 않고 최초/이후 페이지 비교로 빠르게 바꿉니다.(바인딩 기술))
